## ✅ V8.40 定案補充（用戶確認後寫入 SSOT）

1. **P2.5**：大框架**月/季度級**機構數據蒐集與分析，作為**每月/季全部個股重跑世界觀的程序之一**；月/季重跑時若需要的機構籌碼面數據**已是 P5 每日都在蒐集的則直接引用 P5 不需重蒐集**。
2. **高頻蒐集、低頻引用**：P6 已有 20 分鐘級監控；**已高頻率蒐集的資料，低頻率分析就可直接引用，不要重複工作**。
3. **D-3**：個股當天更新「**當天最新 + 歷史（前三天）關聯連結**」的快照 = 統整**全部個股資訊**（新聞+論壇事件集、現股狀態、衍生品狀態…）= **該個股專屬的全方位資訊紀錄統整師**；產出每天這檔股票**最新的狀態報告**，可提到哪些數據有**明顯變化/異常/延續/轉折**；**不負責分析漲跌或策略制定**。
4. **D-4**：蒐集宏觀數據須納入討論串**所有的衍生性以及宏觀指標**；**個股與宏觀 10–12 大類分清楚**（個股由 D-3、宏觀由 D-4）。
5. **快照交付**：**所有快照都有交給後續的決策系統與動態學習系統**。
6. **三層架構**：是**各種工作要搭配正確處理端**的基本原則，**不是指哪個 Phase 做哪個工作**；例如 P5 D-3 中也有程式計算衍生品期權等指標、也有 AI 統整各指標對應的意義。
7. **衍生品與監控**：將目前還沒有的補強跟融合進去，並**確定分工、避免打架與重複工作**。
8. **Daily 最高目標**：建立可被 Weekly 解讀的**時間連續世界模型**；**不寫死「不分析」**— Daily 仍要連接歷史數據、判斷今天數據變動狀態及怎麼統整摘要，只是**不負責分析個股漲跌或策略制定**。
9. **程式數據來源與公式**：**每項該用程式寫的數據來源跟相關公式摘要**要記錄，以免遺漏（見缺口備忘錄/SSOT 附錄）。
10. **Python 數據來源**：若 Python 有**品質更好、更穩、更即時、更乾淨**的數據來源就**改用**；**GAS 時代妥協的產物，有更好的就取代掉，不要把舊的妥協帶到新系統**。
