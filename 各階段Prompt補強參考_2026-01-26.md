## 一、總則

- **憲法五條**：所有 Phase 的 Prompt 設計須符合《核彈投資系統最高總則（憲法）》五條（未來優先、市場視角高於真相、因果鏈完整性、反敘事義務、角色即主力）。
- **角色校準器**：每個 Phase Prompt **最前**加入校準器，強化 AI 扮演的角色與任務（如 P0「未來節點掃描」、P2-2「未來經營狀態預測者」、P3「價格控制者側寫師」等）。
- **未來對齊三問**：每個 Phase Prompt **最後**強制加入三問，強迫模型跳出報告模式，對齊「預測未來」。

## 三、Scout（第一層審查）補強概念

- **定位**：發現「分析框架本身」的盲點與視角偏差，而非補 Checklist。
- **三職權**：Meta-level 框架盲點、權重配置認知偏差、敘事一致但現實不一致。
- **三層次**：
  - **Consistency & Blind Spot Check**（門檻）
  - **Orthogonal Signal Discovery**（關鍵）
  - **Counterfactual World**（高階）
- **輸出結構**：Critique、Discovery 兩方向；固定 JSON 格式輸出：
  - `Most_Probable_Alternative`
  - `Weight_Sensitivity`
  - `Hidden_Signal_Candidates`
  - `Decisive_Tests`
- **不觸發 Deep Auditor**：若 Scout 僅補強、未與 Analyst 明顯分歧，則不觸發 Deep Auditor。

## 四、Deep Auditor（最高層仲裁）補強概念

- **觸發條件**（OR）：分歧、高風險、高影響、自信度不足、輸出不一致、資料異常等（詳見 SSOT 觸發條件清單）。
- **模型**：一律使用 **o3**。
- **輸出**（該 Phase **最終結論，覆寫**）：
  - Rebuild the Causal Chain
  - Scenario Probability Table
  - Action Under Uncertainty
  - What Analyst Missed
- **適用範圍**：除 P0.7、P2-1、P5 Daily D-1～D-4 無 Scout/Deep Auditor 外，其餘有審查者的 Phase 皆套用。
