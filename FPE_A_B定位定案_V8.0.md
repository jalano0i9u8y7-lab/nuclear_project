# FPE_A 和 FPE_B 定位定案 V8.0

**定案日期**：2026-01-16  
**狀態**：✅ 已定案

## ✅ FPE_A 定位（已定案）

### 定義
- **屬於 P2 範圍**
- **由 AI 從白名單搜尋到的公司財報或指引中，提取官方公布的預估 EPS 去計算**
- **更新頻率**：每月/季更新一次

### 實作方式
1. **數據來源**：
   - 美股/台股：財報狗（P2_US_TAIWAN CSE）
   - 日股：buffet code（P2_JAPAN CSE）

2. **提取方式**：
   - AI 從 CSE 搜尋結果中提取「公司財測/來期業績」
   - 從財測/來期業績推導 Forward EPS
   - 獲取當前股價，計算 FPE_A = 當前股價 / Forward EPS

3. **執行時機**：
   - P2_MONTHLY：每月執行
   - P2_QUARTERLY：每季執行

4. **輸出位置**：
   - `Phase2_Output` 表格的 `FPE_A` 欄位

## ✅ FPE_B 定位（已定案）

### 定義
- **更新頻率**：週度（日度太頻繁沒意義，月度太慢無法及時反應市場情緒）
- **屬於 P5 範疇**（因為是週度執行）

### 實作方式
1. **數據來源**：
   - Yahoo Finance Analysis 頁面（爬蟲）
   - 或財報狗（需評估）

2. **收集時機**：
   - **P5 Weekly**：每週執行時收集 FPE_B 數據

3. **整合位置**：
   - 整合到 P5 的市場情緒指標監控中
   - 需要確認 P5 是否已有市場情緒指標監控功能

4. **輸出位置**：
   - 需要新增表格或欄位（待確認）

## 🔍 需要確認的事項

### 1. P5 是否已有市場情緒指標監控？

**檢查結果**：
- ✅ P5 Daily 有新聞收集功能，提到「市場情緒」
- ✅ P5 Weekly 有「市場綜述（Market Analysis）」功能
- ⚠️ 需要確認是否有專門的「市場情緒指標監控」功能

**建議**：
- 如果已有，則整合到現有功能中
- 如果沒有，則新增「市場情緒指標監控」功能

### 2. FPE_B 的數據存儲位置

**選項 A**：新增表格 `MARKET_SENTIMENT_WEEKLY`
- 優點：專門存儲市場情緒數據，結構清晰
- 缺點：需要新增表格

**選項 B**：整合到現有表格（例如：`SMART_MONEY_WEEKLY` 或新增欄位）
- 優點：不需要新增表格
- 缺點：可能與其他數據混淆

**建議**：選項 A（新增表格），因為：
- FPE_B 是市場情緒指標，與籌碼數據性質不同
- 未來可能還有其他市場情緒指標需要存儲

### 3. FPE_B 的數據收集方式

**選項 A**：在 P5 Weekly 中直接收集
- 優點：執行頻率一致（週度）
- 缺點：P5 Weekly 可能需要增加爬蟲邏輯

**選項 B**：在 P5 Daily 中收集，P5 Weekly 讀取
- 優點：數據收集更頻繁（每日），但只取週度快照
- 缺點：P5 Daily 需要增加爬蟲邏輯

**建議**：選項 A（在 P5 Weekly 中直接收集），因為：
- 執行頻率一致（週度）
- FPE_B 不需要每日更新（分析師預估通常不會每天大幅變動）
