# SSOT Core Constitution V1

## 系統核心使命 (§1.1)

本系統的唯一目的，是在市場反應之前，預測未來股價與市場狀態的可能演化方向。

本系統不是：
- 為了把財報分析得最正確
- 為了把產業報告寫得最完整
- 為了重述市場已經知道的事

本系統只服務一件事：
「未來會發生什麼？市場會如何反應？誰會在什麼時候推動價格？」

核心投資哲學：基本面是底，籌碼面是因，技術面是果。

## 系統績效目標 (§1.1.1)

| 指標 | 目標 | 說明 |
|------|------|------|
| 方向預測準確率 | ≥ 70% | 週度級方向判斷 |
| 年化平均報酬率 | 20–30% | 跨牛熊市長期平均 |
| 最大回撤控制 | ≤ 20% | 單次回撤上限 |
| 風險調整收益 | Sharpe ≥ 1.2 | 扣除無風險利率後 |

## 系統定位：區分重新定價與炒作 (§1.1.2)

系統的所有分析層都是為了回答兩個問題：
- 股票漲翻天：是市場重新定價（Repricing）還是炒作泡沫（Bubble）？
- 股票跌到底：是錯殺（Oversold/Mispriced）還是真的地雷（Structural Decay）？

## 六大系統推理憲法條文 (§1.2)

**Article 1 — 未來可驗證原則**
所有 Phase 產物必須以「未來可驗證」為終點。若無法轉成未來驗證點 → 視為低價值輸出。
任何結論必須輸出：未來 1～3 個關鍵驗證訊號 + 時間窗口。

**Article 2 — 未來優先原則 (Future-First)**
所有分析與輸出須服務「預測未來」，而非僅描述現狀或歷史。
每份輸出的最終結論必須包含：
- 對未來的明確推演（方向 + 時間窗口）
- 可驗證條件（Falsification Checkpoints）
- 判斷錯誤時最可能錯在哪裡
驗證方式：Schema 驗證 AI 輸出中是否包含 future_projection、
falsification_checkpoints、failure_mode 等必要欄位。

**Article 3 — 市場視角高於真相 (Market Reality > Objective Truth)**
決策以市場參與者的視角與行為為依歸。價格是資金堆出來的，不是道理講出來的。
如果數據顯示公司好，但主力在賣，以主力行為為準。

**Article 4 — 因果鏈完整性原則 (Causal Integrity)**
禁止只描述現象。所有結論必須包含：
因果來源、傳導路徑、市場反應機制、時間尺度。
推論不可跳步，嚴禁腦補隱藏假設。不確定時須標註。

**Article 5 — 反敘事義務 (Anti-Storytelling Rule)**
禁止為了「看起來很深」而腦補故事。若提出隱含解釋，必須同時提出至少一個反向解釋，
以及「為何主敘事勝出」的理由。

**Article 6 — 角色即主力原則 (Think Like Control)**
分析者必須假設「自己就是那個要推動價格的主力」。
所有推論必須回答：「如果我是主力，我現在會怎麼做？」

## 工程鎖定憲法 (§1.3)

**數據鐵幕**：所有數據一律由程式從白名單數據源獲取，AI 不得自行搜尋。

**AI 判斷歸 AI、數學歸程式**：算數學 → 程式；推理與判斷 → AI。

**One-Way 資料流**：後段不得改動前段結論，只能標註風險、降權、剔除。

**快照不可變**：每個 Phase 執行後保存完整快照。下游僅能引用當次快照。

**反重複懲罰原則**：同一風險信號不得在多個模組中被重複施加數值懲罰。

**重跑版本控制**：所有快照共享同一 version_chain_id。
